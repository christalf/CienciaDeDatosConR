1 Introducción.

La Caja de Herramientas del Científico de Datos, es el primer curso de la serie Especialización en Ciencia de Datos.

En este curso definimos qué es la ciencia de datos y presentamos las herramientas que se utilizarán a lo largo de la serie.

1.1 ¿Qué es la ciencia de datos? El científico de datos y los grandes datos.

La primera pregunta que debe responderse en este curso es ¿qué es la ciencia de datos?

En esencia, ciencia de datos es utilizar datos para contestar preguntas. Esta es una definición demasiado amplia, precisamente porque se trata de un campo muy amplio. La ciencia de datos puede incluir estadística, informática, matemáticas, limpieza y formateo de datos y visualización de datos.

A grandes rasgos, las cualidades que caracterizan a un científico de datos son las de alguien que combina las habilidades del programador de software, el estadístico y el escritor de novelas históricas, para extraer fragmentos de oro ocultos bajo montañas de datos.

Al finalizar los cursos de la especialización, se espera que uno se sienta capacitado para hacer precisamente eso.

Una de las razones del auge de la ciencia de datos en los últimos años es la gran cantidad de datos disponibles en la actualidad y los que se están generando. No sólo enormes cantidades de datos que se recogen acerca de muchos aspectos del mundo y de nuestras vidas, sino que al mismo tiempo tenemos el incremento de la computación de bajo costo. Esto ha creado una tormenta perfecta en la que creamos más datos y las herramientas para analizarlos, mejorando la capacidad de memoria de los computadores, mejores procesadores, más software y ahora también más científicos de datos calificados que pueden ponerlo en práctica y responder preguntas usando estos datos.

Podemos identificar algunas características esenciales de los grandes datos.

La primera es el volumen. Como dice el nombre, los grandes datos implican grandes bases de datos. Estas grandes bases de datos son cada vez más una cosa común. Por ejemplo, digamos que queremos responder a alguna inquietud o pregunta relativa a los videos en línea. Pues resulta que a YouTube se suben aproximadamente 300 horas de video cada minuto. Definitivamente, tendríamos un montón de datos disponibles para analizar. Y se puede ver que habría una gran dificultad para desenmarañar todos esos datos.

Lo cual nos lleva a la segunda cualidad o característica de los grandes datos: la velocidad. Los datos se generan y se recogen ahora más rápidamente que nunca antes. En nuestro ejemplo de YouTube, nuevos datos llegan a cada minuto.

La tercera cualidad es la variedad. En el ejemplo de YouTube, podríamos analizar video o audio, que es un conjunto de datos muy desestructurado, o bien una lista de las longitudes de cada video, visitas o comentarios, que es un conjunto de datos mucho más estructurado para analizar. Existen entonces distitnos tipos de datos que la ciencia de datos estudia.

Volviendo sobre lo que realmente es un científico de datos, podemos decir que la definición más elemental sería que un científico de datos es alguien que utiliza datos para contestar preguntas.

Pero lo más importante es saber lo siguiente: ¿qué destrezas representa un científico de datos? En términos de un diagrama de Venn, podemos identificar a la ciencia de datos en la intersección de tres sectores: conocimientos sustantivos, destrezas de hacking y matemáticas y estadística.

En primer lugar, necesitamos conocimientos suficientes en el área sobre la cual preguntamos para poder formular preguntas y para saber qué tipos de datos son los apropiados para responder a la pregunta. Cuando ya tenemos nuestra pregunta y los datos pertinentes, sabemos, por el tipo de datos, cuál ciencia de datos se encuentra involucrada.

En segundo término, muchas veces los datos deben someterse a limpieza y formateo considerables. Para ello, se requieren destrezas de programación y de hacking.

Por último, una vez que obtenemos nuestros datos, tenemos que analizarlos. Lo cual requiere conocimientos de matemáticas y estadística.

En esta especialización, concentraremos el esfuerzo en conocimiento matemático y estadístico y en destrezas de hacking.

Para las destrezas de hacking, recurriremos al aprendizaje de dos componentes diferentes:

  • Programación en R, la cual le permitirá acceder a los datos, jugar con ellos, analizarlos y graficarlos.
  • Además, nos enfocamos en aprender a cómo averiguar y obtener respuestas a preguntas de programación.

Una de las razones por las que los científicos de datos son tan requeridos es que la mayoría de las respuestas no están ya descrpitas en los libros de texto. Un científico de datos debe ser alguien que sabe cómo encontrar respuestas a problemas nuevos.

Hablando de esa demanda, se necesitan con urgencia personas con destrezas en ciencia de datos y no solo ingenieros en machine learning, científicos de datos e ingenieros de grandes datos.

No sólo tenemos más y más datos, y cada vez más herramientas para recolectarlos, almacenarlos y analizarlos, sino que la importancia de la demanda de científicos de datos cada vez se reconoce más en muchos sectores diversos y no solamente en las empresas y universidades.

Nate Silver es uno de los los científicos de datos o estadísticos más famosos. Es fundador, editor, y director de FiveThirtyEight, un sitio web que utiliza el análisis estadístico y cifras concretas para contar historias acerca de las elecciones, la política, los deportes, la ciencia, la economía y estilos de vida. Utiliza grandes cantidades de datos públicos totalmente gratis para hacer predicciones acerca de una variedad de temas.

Un gran ejemplo de la ciencia de datos en acción es de 2009, cuando los investigadores de Google analizaron 50 millones de términos comúnmente buscados en un período de cinco años y los compararon con datos de la CDC sobre brotes de gripe. Su objetivo era ver si ciertas búsquedas coincidían con brotes de gripe. Uno de los beneficios de la ciencia de los datos que utiliza grandes datos está en que puede identificar correlaciones. En este caso, se identificaron 45 palabras que tenían una marcada correlación con los datos de brote de gripe de CDC. Con estos datos, han sido capaces de predecir brotes de gripe basados únicamente en búsquedas usuales en Google. Sin estas cantidades masivas de datos, estas 45 palabras nunca antes se hubieran predecido.

2 Installing packages from CRAN.

2.1 The install.packages() command.

This is the usual way to install a package from CRAN into R.

  • We just have to put the name of the package between quotation marks.

  • install.packages("ggplot2")

2.2 Installing older versions of a package.

There’s a nice article by Ian Pylvainen that explains this topic in a clear and simple way.

We may need to install an older version of a package due to various reasons:

  • The package might have changed in a way that is incompatible with the version of R we have installed, or with our R code.

  • We may be deploying an application to a location such as shinyapps.io, Shiny Server or RStudio Connect where the environment may not allow us to run the latest version of the package.

We’ll see several methods that we can use to compute this task.

2.2.1 Using the install_version() command from devtools package.

The simplest method is to use the provided install_version() function of the devtools package to install the version we need. For instance:

library(devtools)
install_version("ggplot2", version = "0.9.1", respos = "http://cran.us.r-project.org")

2.2.2 Installing from source with install.packages() command or with R CMD INSTALL command outside of R.

If we know the URL to the package version file we need to install, we can install it from source via install.packages() directed to that URL.

  • We can look for URL of the package version file in the CRAN Package Archive.

  • If we’re on Windows or OS X and looking for a package for an older version of R (R 2.1 or below), we can check the CRAN binary archive.

For example:

packageurl <- "http://cran.r-project.org/src/contrib/Archive/ggplot2/ggplot2_0.9.1.tar.gz"
install.packages(packageurl, repos = NULL, type = "source")

Another way to install from source is using the URL via the R CMD INSTALL command line outside **of R*.

For instance:

wget http://cran.r-project.org/src/contrib/Archive/ggplot2/ggplot2_0.9.1.tar.gz
R CMD INSTALL ggplot2_0.9.1.tar.gz

NOTICE: if we are installing from source, we’ll need to make sure you have the necessary toolchain to build packages from source.

  • On Windows, this may require you to install Rtools.

3 Installing packages from Bioconductor.

3.1 Installing procedure for R versions < 3.5.0.

The way to install Bioconductor’s packages is sourcing the biocLite.R script from Bioconductor’s website, and that will load the bioclite() function that we then use to download the package we want.

For example:

source("http://bioconductor.org/biocLite.R")
biocLite("rhdf5")

3.2 Installing procedure for R versions > 3.5.0.

The new way to install Bioconductor and related packages is through BiocManager package as documented on Bioconductor’s install page.

  • biocLite was a function not a package and had security risks.

Since I’m running R v. 3.6.3, I’ve installed BiocManager following the oficial instructions:

if (!requireNamespace("BiocManager", quietly = TRUE))
        install.packages("BiocManager")

BiocManager::install()

These commands installed Bioconductor 3.10, which has been designed expressly for version 3.6 of R (the current last version of Bioconductor is 3.11).

So, to install specific packages (like “GenomicFeatures” and “AnnotationDbi”), we can use the BiocManager::install() function:

BiocManager::install(c("GenomicFeatures", "AnnotationDbi"))
  • Notice: this function can be use to also install packages from CRAN and even GitHub.
